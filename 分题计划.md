# 分题计划

## 问题分析

当前 `mineru_splitter.py` 存在以下问题：

1. **分题一（纯 Markdown 序号分题）**：依赖正则匹配 `\d+[.、]` 形式，但 MinerU 输出的 Markdown 中题目编号可能被 LaTeX 公式包裹，导致正则匹配失败
2. **分题二（LLM Markdown 分题）**：虽然使用 LLM，但 Markdown 格式丢失了位置信息，图片匹配困难
3. **分题四（LLM 全文分题）**：LLM 直接返回完整题目内容，Token 消耗大，且 LaTeX 源码难以精确切分

**核心问题**：
- 图片与题目匹配不准确，因为缺少位置信息
- 分题依赖文本正则，但 content_list 中每个元素是独立的片段，一道题可能跨多个元素
- 没有利用 content_list 中每个元素的位置信息（page, bbox）

## 解决方案（借鉴 RAGFlow）

RAGFlow 的核心思路：
1. content_list 中每个元素有 `type`（text/image/table/equation）、`page`、`bbox`（位置坐标）
2. 将 content_list 转换为带位置标记的文本序列（`_transfer_to_sections`）
3. 利用位置标记精确匹配图片到题目

## 实施步骤

### 步骤1：删除效果最差的分题方案 ✅
- 删除分题一（`split_by_md_sequential`）
- 删除分题二（`split_by_llm_md`）
- 删除分题四（`split_by_llm_full`）
- 删除 `match_images_to_questions`（旧的图片匹配）

### 步骤2：新建 request分题（借鉴 RAGFlow 的 LLM 边界识别） ✅
- `build_position_text_from_content_list`：将 content_list 转换为带位置标记的线性文本序列
- `split_by_request`：发送给 LLM 识别每道题边界，在带标记的文本中定位切分
- 精确匹配图片到题目（利用 page 和 bbox）

### 步骤3：重命名分题三为 content分题 ✅
- 保留 `split_content_list_ordered` 逻辑
- 增强图片匹配：利用 content_list 中图片的 page 和 bbox 信息
- 图片按页面和位置匹配到最近的题目

### 步骤4：更新前端分题按钮 ✅
- `EngineConfigPanel.tsx`：SPLIT_METHODS 改为仅2个方案
- `PaperUploadPage.tsx`：默认注释更新为 request分题
- 前端只保留两个分题按钮：request分题、content分题

### 步骤5：更新 parse_service 调用 ✅
- `_parse_with_mineru` 采用新分题优先级：
  1. request分题（LLM边界识别 + 位置标记）
  2. content分题（增强版，位置信息精确匹配图片）
  3. LaTeX 锚点分题（回退）
  4. content_list 基础分题（回退）

### 步骤6：更新 API 路由 ✅
- `papers.py` 的 `resplit` 端点：method 范围 1-2
- method=1 → request分题，method=2 → content分题
- 移除旧函数导入和旧方法逻辑
