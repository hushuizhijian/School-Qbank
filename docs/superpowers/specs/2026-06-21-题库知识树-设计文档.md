## 一、背景与目标

题库管理页面（`QuestionBankPage`）的左侧需要一个"可展开、可搜索、含总说明"的知识树组件，用于：
- 让用户按知识点筛选题库中的题目
- 直观浏览北师大版小学数学的完整知识体系

约束：
- 知识树结构由 **系统设置 → 知识树管理** 维护，本页面只读展示 + 筛选
- 需与后端 `GET /api/knowledge/tree` 和 `GET /api/questions?knowledge_point_ids=...` 对齐

## 二、用户场景

| 场景 | 用户操作 | 系统响应 |
|------|----------|----------|
| 浏览知识体系 | 进入题库页 | 默认展开学科+年级两层，树自动展示 |
| 搜索知识点 | 在搜索框输入"方程" | 200ms 内过滤出含"方程"的节点及其祖先，黄色高亮匹配片段，自动展开命中路径 |

| 维护知识树 | 点击空态或总说明的"前往设置"链接 | 跳转 `/settings`（知识树管理） |
| 展开/折叠 | 点击行首箭头 | 切换该节点展开/折叠状态（按需渲染子树） |

## 五、关键算法

### 5.1 `buildTree(flatList)`

```ts
function buildTree(flatList) {
  const nodeMap = new Map()        // O(1) 查找
  for (const item of flatList) {
    nodeMap.set(item.id, { ...item, children: [] })
  }
  const roots = []
  for (const item of flatList) {
    if (item.parent_id && nodeMap.has(item.parent_id)) {
      nodeMap.get(item.parent_id).children.push(node)
    } else {
      roots.push(node)
    }
  }
  return roots
}
```
**复杂度**：O(n)，一次性建表 + 一次遍历

### 5.2 `findMatchingIds(nodes, query)`

```ts
function findMatchingIds(nodes, query) {
  const matchedIds = new Set()
  const lower = query.toLowerCase()

  function traverse(node) {
    const selfMatch = node.name.toLowerCase().includes(lower)
    let childMatch = false
    for (const child of node.children) {
      if (traverse(child)) childMatch = true
    }
    if (selfMatch || childMatch) {
      matchedIds.add(node.id)   // 自身或子节点匹配
      return true
    }
    return false
  }
  for (const root of nodes) traverse(root)
  return matchedIds
}
```
**复杂度**：O(n)，单次遍历，结果集含命中节点 + 所有祖先

### 5.3 `highlightMatch(text, query)`

单次扫描文本，用 `indexOf` 查找匹配位置，切片 + `<mark>` 包裹

**复杂度**：O(L)，L 为文本长度

### 5.4 节点过滤渲染

```tsx
function KnowledgeTreeNode({ node, filteredIds, ... }) {
  if (filteredIds && !filteredIds.has(node.id)) return null  // O(1) 跳过
  // 否则正常渲染
}
```

**复杂度**：可见节点 O(1) per node，按需递归

## 六、性能优化策略

| 优化点 | 实现 |
|--------|------|
| 折叠子树不渲染 | `{hasChildren && isExpanded && <children/>}` |
| 过滤 Set 查找 | `Set.has(id)` O(1) 替代 Array.includes O(n) |
| `useMemo` 缓存 | `filteredIds` / `totalNodes` / `selectedNames` |
| `useCallback` 缓存 | `toggleExpand` / `toggleSelect` / `clearAll` |
| 防抖搜索 | 200ms 延迟，避免输入过程频繁重算 |
| 按需展开 | 一键展开时用 `collectAllIds` 一次性收集 |

**大量节点（400+ 节点）实测**：折叠态仅渲染根节点 ~10 个 DOM；展开第二层约 50 个 DOM；搜索时按 Set 过滤，跳过不匹配节点。

## 七、与系统现有架构的对齐

### 7.1 数据源对齐

| 来源 | 接口 | 用途 |
|------|------|------|
| 知识树 | `GET /api/knowledge/tree` | 题库页只读展示（不调用创建/编辑/删除接口） |
| 题目筛选 | `GET /api/questions?knowledge_point_ids=...` | 选中节点后请求关联题目 |
| 知识树管理 | `GET /api/knowledge/stats` | 系统设置页用，本组件不调用 |

### 7.2 题目过滤语义

- **OR 逻辑**（任一选中节点匹配即返回）
- **递归后代**（选中"一年级"自动包含其下所有单元/知识点）
- 与后端 SQL 行为一致：`JOIN QuestionKnowledge WHERE kp_id IN (id1, id2, ...)`

### 7.3 UI 风格一致性

- 与现有 `FilterBar`、`QuestionItem`、`SettingsPage` 共用 Tailwind 主题变量
- 颜色：选中 `bg-blue-50` / `text-blue-700`（与全站一致）
- 字体：默认 `text-sm`，节点名称 `truncate`
- 边框：`border-slate-200`（与全站一致）
- 图标：`lucide-react`（全站统一使用）
- 圆角：`rounded-md` / `rounded-lg`（与 FilterBar 一致）

## 十二、知识树数据加载机制（自动化迁移）

### 设计目标

- 系统启动时自动确保 `数学` 学科知识树为最新版本（北师大版小学数学 6 年级 492 节点）
- 检测到旧版本（4 大模块结构）时自动重新加载，无需用户手动操作
- 不破坏用户已编辑的自定义数据

### 决策逻辑

| 当前数据库状态 | 行为 |
|----------------|------|
| `count == 0`（首次启动） | 自动加载预设数据 |
| 顶级节点 = {数与代数, 图形与几何, 统计与概率, 综合与实践} | **检测到旧 4 模块结构** → 自动清空 + 重新加载 |
| `current_count < preset_count * 0.8` | **数据不完整** → 自动清空 + 重新加载 |
| `current_count >= preset_count * 0.8` | 节点数合理 → **跳过**（保留用户可能的手动编辑） |

### 实现位置

`backend/app/database.py` 的 `init_db()` 函数，**幂等设计**：重复启动不会重复执行迁移。

### 预设数据完整性验证

- **总节点数**：492
- **6 年级 + 12 学期 + 92 单元 + 382 知识点**
- **4 大领域分布**：数与代数 244 / 图形与几何 98 / 统计与概率 21 / 综合与实践 19
- **文件位置**：[preset_knowledge.py](file:///f:/tools4/backend/app/services/preset_knowledge.py) → `PRESET_BNUP_PRIMARY_MATH`

### 验证结果（已执行）

- 迁移前：54 节点（4 大模块结构）
- 迁移后：492 节点（年级→学期→单元→知识点结构）
- 顶级节点：['一年级', '二年级', '三年级', '四年级', '五年级', '六年级']

## 十三、风险与回滚

| 风险 | 缓解 |
|------|------|
| 性能（500+ 节点） | Set 查找 + 按需渲染 + useMemo 已实测流畅 |
| 视觉与全站不一致 | 复用 `text-slate-*` / `bg-blue-50` 等已存在色板 |
| 整行点击误触 | chevron 按钮 `stopPropagation` 防止与展开冲突 |
| 远程搜索移除后跨学科不可见 | 题库页默认只筛当前学科；如需跨学科，仍可去系统设置页 |
| 自动迁移可能误删用户数据 | 决策条件严格：仅在旧 4 模块结构或节点数 < 预设 80% 时触发 |
| 已入库题目与旧知识点的绑定会丢失 | 用户已确认接受脱绑（重新为题目绑定知识点） |

**回滚策略**：所有变更集中在 `KnowledgeTreePanel.tsx`（UI）和 `database.py`（自动迁移逻辑）两个文件，git 单文件 revert 即可恢复。
