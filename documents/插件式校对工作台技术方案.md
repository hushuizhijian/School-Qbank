# 插件式校对工作台技术方案

## 一、背景与目标

### 1.1 问题

当前校对工作台（`ProofreadingWorkbench.tsx`）直接内嵌约 10 个 AI 组件（`AiMatchKnowledge`、`AiAutoDifficulty`、`AiStandardizeStem` 等），以及 `QualityChecker`、`AiBatchStandardize` 等可选功能。任何一个子组件渲染异常，都会导致整个校对工作台页面白屏崩溃。

### 1.2 目标

将校对工作台改造为"核心 + 插件"架构：

- **核心**：三栏布局 + 题号导航 + 双栏 LaTeX 编辑器 + 入库，保证绝对稳定加载
- **插件**：AI 功能、质量检查、组卷等可选功能，以插件形式挂载，崩溃互不影响

### 1.3 设计原则

- 核心最小化，插件可降级
- 插件崩溃只影响自己，隔离在独立 ErrorBoundary 内
- 插件与核心通过 Props 注入通信，插件内部自治
- 静态清单注册，简单可控

## 三、插件规范

### 3.1 插件元数据

每个插件必须导出以下元数据：

```typescript
// src/types/plugin.ts

/** 挂载位置 */
export type PluginMountPoint =
  | "attribute-panel"   // 中栏属性面板内
  | "editor-bottom"     // 右栏双栏编辑器下方
  | "editor-side"       // 右栏双栏编辑器侧边
  | "toolbar"           // 底部 BatchActionBar 右侧
  | "left-panel"        // 左栏额外区域（预留）
  | "modal"             // 全局弹窗形态（预留）

/** 插件元数据 */
export interface PluginMeta {
  id: string                    // 唯一标识
  mountPoint: PluginMountPoint  // 挂载位置
  priority: number              // 同插槽排序（越小越靠前）
  label: string                 // 显示名称，用于错误占位
}

/** 插件 Props：核心注入的基础数据 */
export interface PluginProps {
  paperId: string                                    // 当前试卷 ID
  currentQuestion: Question | null                    // 当前选中的题目
  onUpdateField: (field: string, value: unknown) => void  // 更新题目字段
  onToggleBank: () => void                            // 切换入库状态
  onRefresh: () => void                               // 刷新题目列表
}
```

### 3.2 插件组件约定

```typescript
// 示例：AiAutoDifficulty 插件改造后
import type { PluginMeta, PluginProps } from "@/types/plugin"

// 1. 导出元数据
export const pluginMeta: PluginMeta = {
  id: "ai-auto-difficulty",
  mountPoint: "attribute-panel",
  priority: 10,
  label: "AI 难度打分",
}

// 2. 组件接收 PluginProps
export default function AiAutoDifficulty({ paperId, currentQuestion, onUpdateField }: PluginProps) {
  // 插件内部完全自治：
  // - 自己的 loading/error 状态
  // - 自己的 API 调用
  // - 自己的 UI 渲染
  // - 通过 onUpdateField 回写结果到核心
}
```

## 四、核心组件设计

### 4.1 PluginErrorBoundary

每个插件被独立的 ErrorBoundary 包裹，渲染崩溃时显示占位 UI。

```typescript
// src/components/plugins/PluginErrorBoundary.tsx

interface PluginErrorBoundaryProps {
  pluginId: string
  pluginLabel: string
  children: ReactNode
}

// 状态：{ hasError, error }
// 正常渲染 → children
// 崩溃时 → 灰色占位卡片:
//   "{pluginLabel} 暂不可用"
//   [重试] 按钮 → clear error → 重新挂载插件
```

### 4.2 PluginSlot

按 `mountPoint` 分组渲染插件，自动按 `priority` 排序。

```typescript
// src/components/plugins/PluginSlot.tsx

interface PluginSlotProps {
  mountPoint: PluginMountPoint
  pluginProps: PluginProps
  plugins: PluginRegistration[]
}

// 逻辑：
// 1. 过滤出匹配 mountPoint 的插件
// 2. 按 priority 升序排序
// 3. 每个插件包裹 <PluginErrorBoundary> + <React.Suspense>
// 4. 渲染插件组件
```

## 六、错误处理策略

### 6.1 三层保护

```
层级 1: PluginErrorBoundary（每个插件独立）
  → 捕获单个插件渲染异常
  → 显示占位 UI + 重试按钮
  → 不影响其他插件和核心

层级 2: PluginHost ErrorBoundary（核心顶层）
  → 捕获核心组件渲染异常
  → 显示整页回退 UI
  → 已存在（src/components/common/ErrorBoundary.tsx）

层级 3: 插件内部 try/catch
  → 网络请求、API 调用异常
  → toast 提示用户，不抛到 ErrorBoundary
  → 保持插件 UI 可见
```

### 6.2 占位 UI 规范

```
┌──────────────────────────────┐
│  ⚠️ AI 难度打分 暂不可用      │
│                              │
│  页面渲染时发生错误，请尝试    │
│  重新加载。                   │
│                              │
│  [查看详情 ▸]                 │
│                              │
│  [🔄 重新加载]                │
└──────────────────────────────┘
```

- 灰色边框卡片，不干扰主 UI
- 重试按钮调用 `setState({ hasError: false })` 触发重新挂载

### 6.3 动态 import 失败

- `React.lazy` + `Suspense` 捕获 `ChunkLoadError`
- 回退到同样的占位 UI，重试重新调用 `loader()`

## 八、风险与边界

### 8.1 不纳入插件的组件

- **AiModelSelector**：保留在核心顶部标题栏，因为它是所有 AI 插件的共享配置源，插件通过 Props 接收 `aiSelection` 而非直接访问
- **DualPaneEditor**：核心编辑器，不可降级
- **QuestionNavigator**：核心导航，不可降级

### 8.2 插件间通信

- 插件之间不直接通信，所有数据通过核心的 `onUpdateField` 回调写入
- 插件如需感知其他插件的行为（如"知识点已更新"），通过核心的 `onRefresh` 回调触发数据重新加载

### 8.3 Zustand Store

- 插件仍可直接 import Zustand store（如 `useSSEStore`），不受插件架构限制
- 推荐做法：优先通过 Props 获取数据，Store 仅用于跨组件共享状态

### 8.4 性能影响

- 所有插件在页面加载时即开始动态 import，不阻塞核心渲染
- `React.lazy` + `Suspense` 确保核心先渲染，插件异步加载
- 插件 JS 文件可利用 Vite 的 code splitting 独立缓存

### 8.5 向后兼容

- 核心组件的 Props 接口不变，AttributePanel 的题型/难度/知识点基础字段逻辑不变
- 插件改造仅影响 AI 组件内部，不改变 API 调用方式
- 现有功能零损失，只改变加载方式

## 九、验收标准

- [ ] 核心组件（三栏布局 + 导航 + 编辑器 + 入库）任何情况下不崩溃
- [ ] 任意单个插件崩溃，不影响核心和其他插件
- [ ] 所有插件崩溃，核心仍可正常使用（排版 + 入库）
- [ ] 插件占位 UI 显示正确标签和重试按钮
- [ ] 重试按钮点击后插件重新加载
- [ ] 现有功能无退化（AI 难度打分、知识点匹配、题干标准化等均正常可用）
