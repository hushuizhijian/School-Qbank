# 题库技术方案

## 一、系统概述

题库系统是 tools4 项目的核心数据模块，负责管理从试卷解析到题目入库、编辑、检索、组卷的全生命周期。系统采用 **FastAPI + SQLAlchemy（后端） + React + Zustand（前端）** 架构，支持题目多维度筛选、知识树管理、TikZ 矢量图渲染、AI 辅助标注等高级功能。

### 1.1 核心能力

- **试卷上传 → MinerU 云端解析 → 分题切分 → 题目入库** 全自动流水线
- **多维度检索**：学科、年级、题型、难度、关键词、知识点、含图/含公式/含表格
- **知识树管理**：三级树形结构，支持题目按知识点分组
- **题目编辑**：LaTeX 源码 ↔ 实时预览双栏编辑器
- **TikZ 渲染**：矢量图代码 → PNG 预览
- **AI 辅助**：智能生成题目解析、优化题干
- **批量操作**：批量入库、批量删除、批量状态更新
- **组卷集成**：从题库选取题目 → 创建作业 → 画布排版 → PDF 导出

### 2.2 Question 模型（题目表）

[backend/app/models/question.py](file:///f:/tools4/backend/app/models/question.py)

```python
class Question(Base):
    __tablename__ = "questions"

    # === 基础字段 ===
    id: str              # UUID 主键
    paper_id: str | None # 所属试卷ID（可空，支持独立题目）
    question_no: int     # 题号

    # === 三区内容 ===
    stem: str            # 题干（Markdown/LaTeX 混合格式）
    options: list        # 选项（JSON 数组）
    answer: str | None   # 答案
    analysis: str | None # 详细解析（V2 新增）

    # === 图表 ===
    images: list         # 图片列表（JSON 数组，支持路径/对象）
    tikz_code: str | None   # TikZ 矢量图代码
    figure_type: str     # 图表类型：screenshot | tikz | latex | none
    boundary: dict | None   # 边界坐标（JSON，含 bbox 信息）

    # === 属性 ===
    question_type: str   # 题型：single_choice | fill_blank | true_false | calculation | solution | ...
    difficulty: str      # 难度：simple | medium | hard
    score: float | None  # 分值（V2 新增）

    # === 状态 ===
    question_status: str # 题目状态：pending | normal | error
    in_bank: bool        # 是否已入库题库
    is_favorite: bool    # 是否收藏
    bank_added_at: datetime | None  # 入库时间

    # === 来源信息（V2 新增）===
    source_paper_name: str | None  # 来源试卷名称
    source_year: str | None        # 出题年份
    source_region: str | None      # 地区
    has_figure: bool               # 是否含图
    has_formula: bool              # 是否含公式
    has_table: bool                # 是否含表格

    # === MinerU 解析产物 ===
    latex_source: str | None  # MinerU 输出的 LaTeX 源码
    knowledge_points: list    # 知识点ID列表（JSON 数组，兼容旧数据）

    # === 时间戳 ===
    created_at: datetime
    updated_at: datetime
```

## 三、题目生命周期

### 3.1 全流程状态机

```
┌─────────┐  上传PDF   ┌──────────┐  MinerU解析  ┌──────────┐
│  pending │ ────────→ │ uploaded │ ────────────→ │ parsing  │
└─────────┘            └──────────┘               └────┬─────┘
                                                       │
                                              ┌────────▼──────┐
                                              │    parsed     │
                                              │ 云端解析完成   │
                                              └───────┬───────┘
                                                      │ 分题切分
                                              ┌───────▼───────┐
                                              │   splitting   │
                                              └───────┬───────┘
                                                      │
                                              ┌───────▼───────┐
                                              │  completed    │
                                              │ 分题完成      │
                                              └───────┬───────┘
                                                      │ 题目入库
                                              ┌───────▼───────┐
                                              │  in_bank=True │
                                              │ 入库题库      │
                                              └───────────────┘
```

### 3.2 题目状态（question_status）

| 状态 | 说明 |
|------|------|
| `pending` | 待审核（默认状态，分题后初始值） |
| `normal` | 正常（已审核通过，可入库） |
| `error` | 异常（解析错误，不参与入库） |

| `solution` | 解答题 | 综合解答 |
| `application` | 应用题 | 实际应用 |
| `short_answer` | 简答题 | 简短回答 |
| `operation` | 操作题 | 动手操作 |
| `matching` | 连线题 | 匹配连线 |

### 4.4 图片智能解析

[backend/app/api/questions.py](file:///f:/tools4/backend/app/api/questions.py#L45-L105)

`GET /api/questions/{id}/image` 接口实现了图片路径的智能解析：

1. 从题目 `images` 字段读取原始路径（支持字符串和对象格式）
2. 结合题目的 `paper_id` 在 `data/papers/` 目录中按优先级查找：
   - 优先 `images/` 子目录
   - 其次根目录平铺
   - 兜底全局 `os.walk` 模糊搜索
3. 按文件扩展名自动推断 `Content-Type`

## 六、知识点管理

### 6.1 知识树结构

知识点采用**三级树形结构**，通过 `parent_id` 自引用实现：

```
一级：数与代数 ────┐
二级：方程与不等式  ├── 三级：一元一次方程
                    ├── 三级：二元一次方程组
                    └── 三级：分式方程
一级：图形与几何
二级：三角形
二级：四边形
```

| `POST /api/questions/{id}/ai-refine` | AI 优化题干 | 调用 LLM 对题干进行规范化改写 |

AI 调用通过 `llm/factory.py` 统一管理，支持多 AI 服务商配置。

## 十、数据库表关系

```
questions  ─────────────────────────────────────────────────────────────
  │
  ├── question_knowledge ─── knowledge_points
  │     (M:N)                    │
  │                              └── parent_id (自引用树形结构)
  │
  ├── homework_questions ─── homework
  │     (M:N 含 sort_order)   │
  │                           └── user_id → users
  │
  └── paper_id → papers
                    │
                    └── user_id → users
```
